CRUD Employee Management System 💻

View Application

This CRUD project was created in a agile method with a group of students. As a product owner, I had to keep track and create user stories our project had. In addition, I was the main Flask API programmer in the group. Using an Object Orientated Method to create classes, we designed an application that kept track of Employees within a business.

With a group of 5 and within 5 weeks, we were able to create an application that allowed employees to be added, edited and deleted within a business. We stored this data using JSON, which I was most experience in, this allowed us to focus on instead of learning new technologies, to find new strategies to improve our skills. My primary focus on this CRUD application was to make sure that I was able to make this application as realistic as possible, such as encrypting sensitive data, and routing pages with an API.

Not only could a single business use the application, but multiple businesses could use this application with Logins containing encryped passwords. Using hashlib, I was able to encryp passwords to make them safe and secure. By storing each businesses/companies data within multiple separate json files, our application could retrieve the corrent information based on the login data. To secure this data, our application used sessions to store session information to store a cache if the user wanted to use a new tab without logging back in. In addition, sessions allowed us to keep track of which information was needed to be shown. If the application was logged out, the session would end, and new credentials would have to be entered again to keep security.

Not only was I able to improve my Javascript techniques, such as using while and for loops, but I was able to understand more about onclick and eventlisteners. Not only did I improve in JavaScript, but I was able to be more creative and understand CSS and HTML in a better way. I used a new element in HTML called svg's, which allowed me to draw the hangman character as letters were guessed incorrectly. I also improved my understanding of JS DOMs, which allowed me to change elements within an HTML to display inline, block, and to also hide elements with none;
